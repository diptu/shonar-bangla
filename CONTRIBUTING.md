# Contributing

Thanks for helping build Shonar Bangla. Here's how.

## Quick Start

```bash
git clone https://github.com/yourusername/shonar-bangla.git
cd shonar-bangla
npm install
npm run dev
```

## Workflow

1. Fork the repo
2. Branch off `main` — `git checkout -b feature/your-feature`
3. Commit — `git commit -m "feat: add district heatmap"`
4. Push — `git push origin feature/your-feature`
5. Open a Pull Request

## Commit Style

We follow [Conventional Commits](https://www.conventionalcommits.org/):

| Type | Use for |
|---|---|
| `feat` | New feature |
| `fix` | Bug fix |
| `docs` | Documentation only |
| `refactor` | Code change, no new behavior |
| `chore` | Tooling, config, deps |

## Before You Submit

- [ ] Code builds — `npm run build`
- [ ] Lint passes — `npm run lint`
- [ ] PR describes **what** and **why**
- [ ] Linked to an issue, if one exists

## Reporting Bugs

Open an [issue](../../issues) with steps to reproduce, expected vs. actual behavior, and screenshots if visual.

## Code of Conduct

Be respectful. Assume good intent. No harassment, ever.
